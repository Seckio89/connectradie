## Findings

### 🔴 HIGH — Supabase: anon-callable SECURITY DEFINER Functions (9)

These functions are callable without authentication (`anon` role) as `SECURITY DEFINER`, meaning they run with elevated DB privileges. Any unauthenticated user can invoke them via `/rest/v1/rpc/<name>`.

> **Context:** The most concerning are the billing-enforcement functions. Note that `12b8148` today added BEFORE UPDATE triggers to lock billing columns — but those are RLS triggers; the RPC callability of these functions is a separate concern.

| Function | Risk | Notes |
|---|---|---|
| `enforce_external_pay_tier()` | HIGH | Payment enforcement logic callable by anon — verify this is intentional |
| `flag_offplatform_payment()` | HIGH | Payment flagging callable by anon — can anon users flag payments? |
| `lock_profile_billing_columns()` | MEDIUM | Billing column lock trigger — callable directly via RPC |
| `lock_tradie_billing_columns()` | MEDIUM | Billing column lock trigger — callable directly via RPC |
| `get_daily_profile_view_count(viewer_uuid uuid)` | LOW | View count — may be intentionally public for profile pages |
| `get_service_worker_details(p_recurring_job_id uuid)` | MEDIUM | Reveals worker details to unauthenticated users |
| `get_team_site_activity(p_since timestamp with time zone)` | MEDIUM | Team activity data leaked to anon |
| `has_user_engagement(user_uuid uuid)` | LOW | Boolean engagement check — low risk but unnecessary for anon |
| `search_businesses_by_name(search_term text)` | LOW | Likely intentional (public tradie search) |

**Remediation for each function** — pick the appropriate fix:

1. **If the function should NOT be public:** `REVOKE EXECUTE ON FUNCTION public.<name> FROM anon;`
2. **If the function is called by authenticated users only:** same REVOKE + ensure `authenticated` role retains EXECUTE
3. **If the function is intentionally public:** no change needed, but document the intent

Example migration:

```sql
-- For billing enforcement functions that should NOT be anon-callable:
REVOKE EXECUTE ON FUNCTION public.enforce_external_pay_tier() FROM anon;
REVOKE EXECUTE ON FUNCTION public.flag_offplatform_payment() FROM anon;
REVOKE EXECUTE ON FUNCTION public.get_service_worker_details(uuid) FROM anon;
REVOKE EXECUTE ON FUNCTION public.get_team_site_activity(timestamptz) FROM anon;
REVOKE EXECUTE ON FUNCTION public.lock_profile_billing_columns() FROM anon;
REVOKE EXECUTE ON FUNCTION public.lock_tradie_billing_columns() FROM anon;
```

> Supabase docs: https://supabase.com/docs/guides/database/database-linter?lint=0028_anon_security_definer_function_executable
