## 🟡 MEDIUM — Performance Issues

### 3. RLS Init Plan — `quote_templates.qt_owner_all`

The `qt_owner_all` policy calls `auth.uid()` directly, which Postgres re-evaluates **per row**. Replace with `(select auth.uid())` to evaluate once per query.

**Fix:**
```sql
-- New migration: 20260719000001_fix_qt_owner_all_rls_initplan.sql
DROP POLICY IF EXISTS qt_owner_all ON public.quote_templates;
CREATE POLICY qt_owner_all ON public.quote_templates
  FOR ALL
  TO authenticated
  USING ((select auth.uid()) = tradie_id)
  WITH CHECK ((select auth.uid()) = tradie_id);
```

### 4. Multiple Permissive Policies

Two tables have duplicate RLS policies for the same role+action, causing double policy evaluation on every query:

- **`quote_templates`**: Both `"Users can manage own quote templates"` and `qt_owner_all` fire for SELECT/INSERT/UPDATE/DELETE by `authenticated`. The old policy should be dropped once `qt_owner_all` is confirmed correct.
- **`custom_task_suggestions`**: Both `cts_admin_read` and `cts_read_approved` fire for SELECT by `authenticated`.

### 5. Unindexed Foreign Keys

`custom_task_suggestions` is missing indexes on:
- `custom_task_suggestions_reviewed_by_fkey` (column 9)
- `custom_task_suggestions_submitted_by_fkey` (column 2)

**Fix:**
```sql
CREATE INDEX IF NOT EXISTS idx_custom_task_suggestions_reviewed_by
  ON public.custom_task_suggestions (reviewed_by);
CREATE INDEX IF NOT EXISTS idx_custom_task_suggestions_submitted_by
  ON public.custom_task_suggestions (submitted_by);
```
