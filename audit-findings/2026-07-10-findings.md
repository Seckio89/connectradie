### Multiple Permissive RLS Policies (same table/role/action)

Each overlapping policy pair is evaluated for every query — hurts performance:

| Table | Role | Action | Overlapping Policies |
|-------|------|--------|---------------------|
| `business_team_members` | authenticated | INSERT | "Business owner can insert team members", "Members can request to join a team" |
| `invoices` | authenticated | SELECT | "Billed users can view their invoices", "Users can view accessible invoices" |
| `invoices` | authenticated | UPDATE | "Billed users can update their invoices", "Users can update own invoices" |
| `jobs` | authenticated | DELETE | "Tradies manage jobs for own contacts", "Users can delete own jobs" |
| `jobs` | authenticated | INSERT | "Clients can create jobs", "Tradies manage jobs for own contacts" |
| `jobs` | authenticated | SELECT | "Tradies manage jobs for own contacts", "Users can view relevant jobs" |
| `jobs` | authenticated | UPDATE | "Tradies manage jobs for own contacts", "Users can update relevant jobs" |
| `recurring_jobs` | authenticated | SELECT | "Assigned worker can view service", "Users can manage their own recurring jobs" |
| `recurring_sessions` | authenticated | SELECT | "Assigned worker can view service sessions", "Users can manage relevant sessions" |
| `site_visit_events` | authenticated | SELECT | "Clients read site visit events for their jobs", "Tradies read own site visit events" |

**Fix:** Consolidate overlapping policies into single policies using `OR` conditions.
**Docs:** https://supabase.com/docs/guides/database/database-linter?lint=0006_multiple_permissive_policies
